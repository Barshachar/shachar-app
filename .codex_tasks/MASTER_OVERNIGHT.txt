TITLE: Overnight full enablement for Multi-Vendor B2B/B2C + In-App Agent (Supabase + App) — GPT-5

SCOPE (from “תכנון אפליקציה מולטי-וונדור” + “אפליקציית B2B מושלמת”):
- Multi-Vendor B2B:
  • ישויות: vendors, vendor_users, product_vendors, vendor_settings, payouts.
  • הפרדה/אבטחה: RLS רב-דיירים (vendor_slug), כל ונדור רואה/מנהל רק את שלו.
  • קטלוג לכל ונדור, SKU mapping, MOQ/מינימום הזמנה.
  • price lists לפי לקוח/קבוצות/וונדור; טירים/הנחות.
  • זרימות B2B: Quick-Order (SKU+Qty), Quote→Order, Approvals, Budget/credit limit, Returns/Credit notes.
  • “החנות שלי” לוונדור: KPIs, מוצרים מובילים, מלאי נמוך, יצוא CSV; payouts לתקופות.
- In-App AI Assistant:
  • Agents SDK / Responses API.
  • כלים: search_products, price_for_customer, create_order, update_order_status, get_vendor_kpis, File Search אם יש.
  • Guardrails (לדוגמה: הנחה >X% דורשת אישור).
  • UI צ’אט מיקרו-פרונטנד (Next.js) משובץ באפליקציה (WebView/iframe), עברית.
- B2C Playbook (Flatsome/WordPress) — תיעוד בלבד:
  • דף בית: מוצרים חמים/חדשים; כפתור WhatsApp; ללא כפתורי שיתוף בעמוד מוצר.
  • פרפורמנס: LCP, Critical CSS, Fonts, Cache; WP-CLI ייצוא/ייבוא; Staging→Prod.

CONSTRAINTS:
- ללא DROP; רק CREATE IF NOT EXISTS / ALTER ADD.
- לא מכניסים סודות ל-git; לעדכן .gitignore.
- עבודה בענף codex/overnight-{{today}}; קומיטים קטנים; לוגים לכל שלב.
- Python 3.11 אם קיים; אחרת system python3; eval_type_backport לתאימות py<3.10.

STEPS:

1) Repo & Env
- יצירת/מעבר לענף codex/overnight-{{today}}.
- .gitignore ל-.env, .venv, node_modules, build, logs, dist, .dart_tool.
- rebuild .venv; pip install -r requirements.txt + eval_type_backport.
- ודא agent/__init__.py; bin/run_agent.command יריץ "python -m agent.run_agent".

2) Agent Reliability (fallbacks)
- agent/order_agent.py: הוסף "from __future__ import annotations".
- price_for_customer: אם RPC נכשל → קרא price מ-products (REST); החזר {"sku","customer_id","price","currency":"ILS"}.
- create_order: אם RPC נכשל → POST orders + order_items (REST) עם price מ-products; החזר {"order_id","customer_id","total","items"}.
- agent/run_agent.py: תמיכת --once/--prompt לסמוק לא-אינטראקטיבי; שגיאות ברורות בעברית.

3) Supabase Schema (מינימלי, לא הורס)
- צור אם חסר:
  vendors(slug pk, name, status default 'active', created_at),
  vendor_users(vendor_slug fk, user_id text, role text default 'member', primary key (vendor_slug,user_id)),
  product_vendors(sku text, vendor_slug text, primary key (sku,vendor_slug)),
  vendor_settings(vendor_slug fk, settings jsonb, updated_at timestamptz),
  payouts(id uuid pk, vendor_slug fk, period_start, period_end, amount numeric, currency text default 'ILS', status text default 'draft', created_at).
- אינדקסים; RLS stubs לפי vendor_slug; ENABLE RLS.
- Seed אם צריך: vendors('ashachar'), categories('uncategorized'), brands('generic').
- Upsert demo product SP-NTM-32 price 12.90 slug sp-ntm-32 mapped to 'ashachar'.

4) App (B2B “מושלם”)

- “החנות שלי” לוונדור: KPIs יומיים, מוצרים מובילים, מלאי נמוך, יצוא CSV.
- Role-based UI (buyer/manager/vendor/admin); מסכי Approvals להנחות חריגות.
- בדיקות לרפו/שירותים (mock Supabase client); lint (dart analyze / npm test לפי הפרויקט).
